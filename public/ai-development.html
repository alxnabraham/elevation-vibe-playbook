AI-Assisted Development

How to keep your branch in sync when using AI coding tools. This guide covers the critical pull-before-coding workflow, tool-specific setup, and how to review AI-generated changes.

1. Why This Matters

The #1 cause of broken builds
When an AI assistant works on stale code (code that is behind the latest version), it generates changes against an outdated baseline. When you push those changes, they conflict with work others have already merged. This wastes everyone's time and can break the demo site.

AI coding tools like Copilot, Cursor, and Claude Code can modify dozens of files in a single session. If your branch is out of date, every one of those changes is a potential conflict. The fix is simple: always sync before you start.

3. Before Every AI Session

Do this EVERY time — no exceptions
This is the same Step 1 from the Daily Workflow guide. It applies every time, but is especially critical before handing control to an AI tool that may change many files at once.

Open VS Code / Cursor / Windsurf and confirm you are on your dev-[yourname] branch.

Check the bottom-left corner of the editor. If it says demo or staging, switch to your branch first.

Open the terminal and run:

git pull origin staging

This downloads all the latest approved code from the staging branch and merges it into your branch.

Check for errors. If you see a "merge conflict" message, stop immediately and message Alan on Slack with a screenshot. Do NOT start an AI session with unresolved conflicts.

Now start your AI session. Your code is up to date and safe to work on.

Quick check
If git pull origin staging says "Already up to date." — you are good to go. If it shows a list of file changes — that means new code was downloaded, and you are now current.

4. Working with Specific AI Tools

Each tool works slightly differently, but the sync workflow is the same for all of them. Here is how to use each one.

GitHub Copilot (in VS Code)

Copilot is built into VS Code and has an Agent mode that can create and edit files for you.

  • Open Copilot Chat: Ctrl + Alt + I (Windows) or Cmd + Option + I (Mac)
  • Set the chat mode to Agent for multi-file changes
  • For best results, select Claude Sonnet 4.5 or Claude Sonnet 4.6 in the model picker, or leave it on Auto
  • If using the ElevationAI custom agent, select elevationai-app in the agent picker (see Custom Agent Setup)

Cursor

Cursor is a VS Code fork with built-in AI capabilities.

  • Use Composer or Agent mode for multi-file changes
  • Git commands work identically to VS Code — same terminal, same Source Control panel
  • Always pull before starting a Composer session

Claude Code (CLI)

Claude Code runs in the terminal as a command-line assistant.

  • Open your terminal and navigate to the project directory
  • Run git pull origin staging before starting a claude session
  • Claude Code can run git commands, but pull manually first to verify no conflicts exist
  • When done, review changes with git diff before pushing

Windsurf

Windsurf is another VS Code-based editor with AI integration.

  • Use Cascade or Agent mode for complex tasks
  • Same Git workflow as VS Code — pull first, push after
  • Check bottom-left for branch name before starting

5. After the AI Finishes

Always review before pushing
AI assistants sometimes modify files you did not ask them to change. Always review the full list of changed files in the Source Control panel before committing.

Review all changed files. Click the Source Control icon (Ctrl+Shift+G / Cmd+Shift+G) and look through every file that was modified. Watch for:

  • Files you did not expect to be changed
  • Changes to package.json or package-lock.json (the AI may have added unnecessary packages)
  • Changes to .env or any config files

Test your changes. If you can run the app locally, start the dev server and check that things work:

npm run dev

If local development is not set up, you can push and check the Vercel preview instead.

Commit with a clear message. Describe what you (or the AI) changed:

git add .
git commit -m "Updated calendar layout and fixed sidebar spacing"
git push

Check your preview URL. Wait 1–3 minutes for Vercel to rebuild, then refresh your preview URL to verify the changes look correct.

6. Custom Agent Setup (Copilot)

We have a custom ElevationAI agent for GitHub Copilot that enforces our framework rules, design system, and code patterns. Setting it up takes 2 minutes and makes Copilot generate much better code.

Create the agent file. In your project root, create the folder .github/agents/ if it does not exist, then create a file called vibeapps.agent.md inside it.

.github/agents/vibeapps.agent.md

Download the agent content. Get the latest agent file from the team docs and paste it into vibeapps.agent.md. Save the file.

Select the agent in Copilot Chat. Open Copilot Chat, set mode to Agent, and choose elevationai-app from the agent picker.

What does the custom agent do?
It pre-loads framework rules, protected files, tech stack constraints (Next.js, React, Tailwind, Supabase), API conventions, security requirements, and design system tokens. This means Copilot generates code that fits our codebase instead of generic code.

7. Common AI Pitfalls

ProblemWhat HappenedHow to Fix
Merge conflicts after pushing You forgot to pull before the AI session Message Alan. Next time, always run git pull origin staging first.
AI modified unexpected files AI tools sometimes "fix" things you did not ask about Review Source Control panel. Revert unwanted changes by right-clicking a file → Discard Changes.
AI changed package.json AI added packages that are not needed or changed versions Revert with git checkout package.json package-lock.json, then run npm install.
AI changed environment or config files AI modified .env, next.config, or similar Revert those specific files. Never commit .env files.
AI created files outside the project Some tools create temp files or files in wrong directories Delete the stray files. They will not deploy.
Build fails on Vercel after push AI introduced a syntax error or import issue Check the Vercel build log for the error. Ask the AI to fix it, then push again.
AI made changes on the wrong branch You were on demo or staging instead of your branch Do not panic. Message Alan immediately. Always check bottom-left of VS Code.

8. Example Prompts for AI Assistants

Here are prompts that work well with our codebase. Be specific about what you want.

Good prompts

Create a new dashboard page at /dashboard that:
- Uses the Elevation design system tokens
- Shows a welcome message with the user's name
- Displays metrics in cards using the existing MetricsGrid component
- Uses Tailwind CSS with our theme variables
Fix the background color on the space dashboard to match
the theme. Use our CSS variables (--background, --surface)
instead of hardcoded colors.
Add a settings page that uses the existing PlatformAppLayout
shell with sidebar navigation. Follow the pattern in the
existing workspace page.

Tips for better results

  • Reference existing code: Say "follow the pattern in [existing file]" or "similar to the workspace page"
  • Mention the design system: Say "use Elevation design tokens" or "use our CSS variables"
  • Be specific: Instead of "make a page", say "create a page at /dashboard that shows X in Y layout"
  • Iterate: Start simple, then add features: "Now add a search bar" or "Make it responsive"
  • Ask for explanations: "Explain what this function does" or "Why is this not working?"
