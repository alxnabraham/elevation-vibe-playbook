Environment Setup

One-time setup for your development environment. This covers installing tools, cloning the project, configuring environment variables, and setting up authentication. Takes about 15–20 minutes.

You only need to do this once
After completing these steps, jump to the Daily Workflow for your everyday process.

1. Install Required Tools

Node.js

Node.js runs JavaScript on your computer. The project needs it to work.

Download the LTS version from nodejs.org.

Run the installer. Click Next on each screen — the defaults are fine.

Verify it works. Open a terminal and run:

node --version
npm --version

You should see version numbers like v20.x.x and 10.x.x.

VS Code (or Cursor / Windsurf)

VS Code is our primary code editor. Cursor and Windsurf are VS Code forks that work identically.

Download VS Code from code.visualstudio.com.

Install and open it.

Install these recommended extensions:

  • GitHub Copilot — AI-assisted coding
  • ES7+ React/Redux/React-Native snippets — React autocomplete
  • Prettier — code formatting
GitHub Copilot access
You need an active GitHub Copilot subscription. If your account does not have Copilot enabled through the organization, contact your GitHub org admin.

Git

Git tracks file changes and lets you download code from GitHub.

Windows: Download from git-scm.com/downloads/win. Choose the 64-bit installer and click Next through each screen.

Mac: Open a terminal in VS Code and run:

xcode-select --install

Click Install and wait. If you see "already installed", Git is ready.

Verify:

git --version

You should see git version 2.x.x. Restart VS Code after installing Git.

2. Clone the Project

This downloads the project code to your computer.

Open VS Code.

Press Ctrl + Shift + P (Windows) or Cmd + Shift + P (Mac) to open the Command Palette.

Type Git: Clone and select it.

Paste the repo URL from the Slack #ws-apps channel.

Choose a save location (Documents or Desktop).

Click Open when prompted.

Install dependencies. Open the terminal and run:

npm install
Getting "Unauthorized" or "Access Denied"?
Your GitHub account does not have repository access yet. Message Alan on Slack to get added to the Elevation AI organization.

3. Environment Variables

The project needs configuration to connect to our backend services.

In the Slack #ws-apps channel, find the pinned message with the environment variables.

In the project root, create a new file called .env.local (note the dot at the beginning).

Copy and paste the contents from Slack into this file and save.

Environment Variable Reference

VariableDescriptionRequired
AUTH_API_URLPlatform API URL for authenticationYes
AUTH_API_VERSIONAPI version headerNo

Never commit .env files
.env.local contains private keys. It is already in .gitignore but do not rename it or share it outside the team.

4. Authentication — Cookie Setup

Our app uses cookie-based authentication. You need to copy auth cookies from the platform to your development environment.

Choose your environment

EnvironmentURLAuth CookieUser Cookie
Developmentdevagents.elevationai.comelevationai-dev-authelevationai-dev-user
Stagingstagagents.elevationai.comelevationai-stag-authelevationai-stag-user

Copy cookies to your preview URL

Open your browser and go to the platform URL (e.g., devagents.elevationai.com). Sign in with your @elevationai.com Google account.

Open Developer Tools: F12 or Ctrl + Shift + I (Windows) / Cmd + Option + I (Mac).

Click the Application tab, expand Cookies, and click the current site.

Find your auth cookies (see table above). For each cookie:

  • Double-click the Domain column
  • Change it to your Vercel preview URL domain (just the domain, no https://)
  • Press Enter to save

Open your preview URL in a new tab and refresh. You should see the app.

For localhost development
If working locally, change the cookie domain to localhost instead of the Vercel URL. The cookie may disappear from the original domain tab — that is expected. It moved to localhost.

5. Alternative: Console Method

If the cookie domain edit does not work, you can set cookies via the browser console.

Sign in to the platform and copy the Value of each auth cookie from the Application tab.

Open your target URL (localhost or preview URL) in a new tab.

Open Developer Tools, go to Console, and run:

document.cookie = "elevationai-dev-auth=YOUR_AUTH_VALUE; path=/; domain=localhost";
document.cookie = "elevationai-dev-user=YOUR_USER_VALUE; path=/; domain=localhost";

Replace YOUR_AUTH_VALUE and YOUR_USER_VALUE with the actual cookie values. For Vercel preview URLs, replace localhost with the preview domain.

Refresh the page.
