Reference & Troubleshooting

How everything connects, AI development tips, design system reference, and solutions to common problems.

1. The Big Picture

Git tracks every change to our code (like Google Docs version history). GitHub is where the code lives online (like a shared Drive folder). Vercel turns the code into a live website — every push to GitHub triggers an automatic rebuild.

You edit code in VS Code
Push to your branch on GitHub
Vercel auto-builds your preview
You see it live!

How branches and previews connect

Each branch has its own Vercel preview URL that updates automatically when you push. The flow for getting changes into the live demo site is:

Your dev-* branch
Verify on your preview
sandbox (shared testing)
demo (investors see this)
Why this matters
This flow ensures nobody's broken code reaches the demo site. You verify on your own preview first, then Admin merges to sandbox, and finally to demo.

2. Working with AI Tools

GitHub Copilot in VS Code

  • Open Copilot Chat: Cmd+Option+I (Mac) or Ctrl+Alt+I (Windows)
  • Set mode to Agent for multi-file changes
  • Select elevationai-app in the agent picker to use our framework rules
  • Use Claude Sonnet 4.5/4.6 in the model picker for best results
  • Describe what you want to build in plain English — Copilot handles the code
  • When done, tell Copilot: "Push all my changes to GitHub"

5. Design System Essentials

Colors

Always use Elevation tokens from globals.css — never hardcode hex values.

TokenUsage
--backgroundPage background
--foregroundPrimary text
--primaryBrand color (Elevation Blue #0e62fd)
--cardCard backgrounds
--borderBorders and dividers
--mutedSubtle backgrounds
--destructiveErrors, delete actions

Typography

FontVariableUsage
Instrument Sans--font-sansAll UI text

Theming

The app supports Light, Dark, and System modes. All color tokens adapt automatically. When building, use the token names — they handle both themes.

7. Troubleshooting: Auth & Access

"Access Denied" or blank screen

  • Sign in at devagents.elevationai.com with your @elevationai.com Google account
  • Try an Incognito/Private window
  • If using Safari — switch to Chrome (Safari has stricter cookie policies)
  • If none of this works — message Admin. Jay may need to whitelist your preview URL.

"Not Authenticated" error

  • Your session likely expired. Go to the platform, log in again.
  • If working locally, re-copy auth cookies (see Slack #ws-apps for instructions).

8. Troubleshooting: Git & Branches

"Merge conflict" error

Do NOT try to fix this yourself
Take a screenshot, message Admin on Slack. Incorrect conflict resolution can break the codebase for everyone.

I pushed to the wrong branch

Don't panic. Message Admin immediately with which branch and what you changed. This is fixable.

I'm working on stale code

Run git pull origin staging. This should be the first thing you do every time you open VS Code.

"Unauthorized" when cloning

Your GitHub account doesn't have access. Message Admin to get added to the Elevation AI organization.

9. Troubleshooting: Vercel & Previews

Preview shows old content

  • Wait 2–3 minutes after pushing for Vercel to rebuild
  • Hard-refresh: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
  • Check the Vercel dashboard for build errors

"Does not have contributing access"

  • Your Vercel account isn't on the Elevation AI team
  • Message Admin for a team invite
  • Your Vercel email must match your GitHub email

Build failed

  • Click the failed deployment in Vercel to see the build log
  • Common causes: syntax error, missing import, broken component
  • Fix locally, push again

10. Troubleshooting: AI Assistant Issues

AI modified files I didn't ask about

Source Control panel → right-click the unwanted file → Discard Changes. Always review before committing.

AI code broke the build

Check Vercel build log, paste the error into your AI chat: "Fix this build error." Push the fix.

AI isn't using our design system

Tell it: "Use Elevation design tokens from globals.css" or "Follow the pattern in [existing file]." Make sure the agent file is installed.

Changes are only on my computer — preview hasn't updated

You need to push. Run: git add . && git commit -m "message" && git push. Or tell your AI: "Push all my changes to GitHub."
