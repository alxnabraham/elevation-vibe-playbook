Starter Kit & Design System

Reference guide for the Elevation platform architecture, design tokens, available components, and how to extend the codebase. This is your map to the EI V2 project.

1. What is the Starter Kit?

The Elevation Starter Kit is a Next.js 14 application framework that provides the complete scaffolding for building apps on the Elevation platform. It includes:

  • Platform shell and navigation foundations
  • Page templates and reusable platform components
  • Typed action and insight primitives
  • Design system tokens and global theme primitives
  • Working examples for every component type

Tech stack: Next.js 14, React 18, TypeScript, Tailwind CSS 4, Radix UI, Recharts, Supabase.

Quick Start

npm install
npm run dev

Open http://localhost:3000 to see the starter app.

3. Colors

The app uses Elevation color tokens defined in src/app/globals.css. Always use these tokens instead of hardcoded hex values.

Primary tokens

TokenUsage
--backgroundPage background
--foregroundPrimary text
--primaryBrand color (Elevation blue)
--mutedMuted backgrounds
--cardCard backgrounds
--surface-*Surface layers (elevated, sunken)
--borderBorders and dividers
--destructiveError states, delete actions
Use tokens, not hardcoded colors
If you or your AI assistant writes bg-blue-500 or #4361EE directly, replace it with the appropriate token like bg-primary or var(--primary). This ensures consistency across light and dark mode.

5. Corner Radius

The app supports 6 corner radius presets applied globally via the GlobalCornerRadiusProvider.

PresetValueStyle
Sharp0pxNo rounding
Compact4pxSubtle rounding
Tight6pxSlightly rounded
Balanced8pxModerate rounding
Standard (default)10pxStandard rounding
Soft14pxFully rounded feel

Presets are defined in src/lib/corner-radius-presets.ts. Use the tokens --radius and --radius-sm in your CSS.

6. Light & Dark Mode

The app supports Light, Dark, and System themes via next-themes. The theme switcher is in the header.

  • Theme is applied via a class attribute on <html>
  • All color tokens automatically adapt between light and dark mode
  • Logo SVGs should invert in dark mode (use filter: invert(1))
  • The ThemeProvider is configured in src/app/layout.tsx

10. For AI Coding Agents

When using an AI assistant in this codebase, keep these rules in mind:

  • Prefer extending existing primitives over creating parallel systems
  • Use semantic tokens from globals.css — avoid ad-hoc colors or one-off styling
  • Preserve theming — light/dark/system toggle and corner-radius presets must keep working
  • Reference the canonical API from @/platform for shell, navigation, templates, and actions
  • Keep route structure stable unless explicitly asked to change behavior
Master prompt for AI agents
For one-shot onboarding of an AI agent, use the master prompt "Start a New App (Starter Kit)" found in design-system-contract/prompt-library.md. The agent will return a setup summary and next steps.

11. How to Extend

Update src/starter/app-manifest.ts with your app IDs, navigation items, and entity routes.

Replace sample routes in src/app/ with your domain-specific pages.

Reuse EntityPage and WorkspacePage templates to maintain the Data → Insights → Actions flow.

Register production action handlers via registerPlatformActions and your execution adapter.
