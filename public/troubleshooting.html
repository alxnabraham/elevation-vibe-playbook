Troubleshooting

Solutions for the most common problems. Find your issue by category, follow the fix, and if nothing works — message Alan with a screenshot.

1. Authentication & Access

"Access Denied" or blank screen on the app

  • Make sure you are signed in at devagents.elevationai.com with your @elevationai.com Google account.
  • Check that your cookies are set correctly (see Cookie Setup).
  • Try opening the site in an Incognito / Private window and signing in fresh.
  • Clear your browser cookies for the site and redo the cookie setup.
  • If none of this works, message Alan. Jay may need to add your preview URL to the backend whitelist.

"Not Authenticated" error after starting the app

  • Your cookies have likely expired. Go back to the platform, log in again, and repeat the cookie-copy process.
  • Make sure the Domain on your cookies is exactly localhost (no protocol, no port) for local dev, or your exact Vercel preview domain for preview URLs.

Cookies disappear or keep getting logged out

  • This is common in Safari. Safari has stricter cookie policies. Try using Chrome instead.
  • If using Chrome, make sure you are not in a mode that clears cookies on close.
  • Platform sessions can expire after inactivity. Keep a platform tab open, or repeat the cookie process when needed.

Cookies not appearing on localhost

  • Make sure the cookie domain is exactly localhost — no http://, no :3000.
  • The cookie may "disappear" from the original domain tab — that is expected. It moved to localhost.
  • Try the console method as an alternative.
  • Refresh the page after setting cookies.

2. Git & Branches

"Merge conflict" error when pulling

This means someone else changed the same file you changed.

Do NOT try to fix merge conflicts yourself
Take a screenshot of the error, message Alan on Slack, and he will help resolve it. Incorrect conflict resolution can break the codebase for everyone.

I accidentally pushed to the wrong branch

  • Do not panic. This is fixable.
  • Message Alan immediately with which branch you pushed to and what changes you made.
  • Prevention: Always check the bottom-left corner of VS Code before doing any work. It should say dev-[yourname].

I am working on stale / outdated code

  • Run git pull origin staging to download the latest approved code.
  • This should be the first thing you do every time you open VS Code. See Daily Workflow Step 1.

"Unauthorized" or "Access Denied" when cloning

  • Your GitHub account does not have access to the Elevation AI organization yet.
  • Message Alan to get added.

I do not see my branch in VS Code

  • Click the branch name in the bottom-left corner.
  • Select "Create new branch from...", type dev-[yourname], and select demo as the source.

3. Vercel & Preview URLs

"Does not have contributing access to the project"

  • Your Vercel account is not linked to the Elevation AI team.
  • Go to vercel.com and check if "Elevation AI" appears in the top-left team dropdown.
  • If not, message Alan to resend the team invite. Accept it from your email.
  • Important: The Vercel account email must match the GitHub account that pushes code. If you signed up with a different email, that is likely the issue.

My preview URL shows old content

  • Wait 2–3 minutes. Vercel rebuilds after each push and it takes time.
  • Hard-refresh your browser: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac).
  • Check Vercel's dashboard — look for the latest deployment under your branch. If it says "Building", wait. If it says "Error", check the build log.

Build failed on Vercel

  • Click the failed deployment in the Vercel dashboard to see the build log.
  • Common causes: syntax error in code, missing import, broken component.
  • Fix the error locally, commit, and push again.
  • If you cannot understand the error, screenshot the build log and message Alan.

"Deployment not found"

  • Your branch may not have been pushed to GitHub yet, so Vercel has no deployment.
  • Push your code: git push.
  • If you already pushed, check that you pushed to the correct branch name.

4. AI Assistant Issues

AI modified files I did not ask it to change

  • Open the Source Control panel (Ctrl+Shift+G / Cmd+Shift+G).
  • Right-click on any file you do not want changed and select Discard Changes.
  • Always review all changed files before committing.

AI changed package.json or config files

  • Revert those specific files:
git checkout package.json package-lock.json
npm install
  • Never commit changes to .env files.

AI created merge conflicts

  • This happens when you forgot to pull before the AI session.
  • Message Alan to resolve the conflict.
  • Prevention: Always run git pull origin staging before starting any AI session. See Before Every AI Session.

AI code broke the Vercel build

  • Check the Vercel build log for the specific error.
  • Ask the AI to fix it: paste the error and say "Fix this build error."
  • Push the fix.

AI is not using our design system

  • Tell the AI explicitly: "Use Elevation design tokens from globals.css" or "Follow the pattern in [existing file]."
  • Set up the custom Copilot agent to enforce framework rules automatically.

5. Development Environment

"Cannot find module" errors

  • Run npm install to reinstall dependencies.
  • If that does not work, delete node_modules and reinstall:
rm -rf node_modules
npm install

Port 3000 already in use

  • Another process is using port 3000. Run on a different port:
npm run dev -- -p 3001

.env changes not taking effect

  • After changing .env or .env.local, you must restart the dev server.
  • Stop the server (Ctrl+C) and run npm run dev again.

App loads but looks broken (no styles)

  • Make sure npm install completed without errors.
  • Try deleting .next folder and restarting: rm -rf .next && npm run dev.
  • Check that you have not accidentally modified globals.css or layout.tsx.
